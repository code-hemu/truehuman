# React Example

Integrating TrueHuman into a React application is done through custom hooks that wrap the `analyze` function inside a `useEffect` and expose the result alongside a loading state. The examples below cover both the basic usage pattern and a more advanced configuration with plugins.

## Basic Bot Detection Hook

The simplest integration uses `analyze()` directly with no additional configuration. Call it once on mount inside a `useEffect`, store the result in local state, and expose it to the rest of your component tree.

```tsx
import { useEffect, useState } from "react"
import { analyze, type AnalyzeResult } from "truehuman"

export function useBotDetection() {
  const [result, setResult] = useState<AnalyzeResult | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    setResult(analyze())
    setLoading(false)
  }, [])

  return { result, loading }
}
```

### Usage

Drop the hook into any component that needs access to the detection result. The `loading` flag is `true` until `analyze()` has finished and the result has been stored, making it straightforward to conditionally render UI based on whether detection is complete.

```tsx
function App() {
  const { result, loading } = useBotDetection()

  if (loading) return <p>Analyzing...</p>

  return (
    <div>
      <p>Score: {result?.score}</p>
      <p>Is Bot: {result?.isBot ? "Yes" : "No"}</p>
    </div>
  )
}
```

The `result` object conforms to the `AnalyzeResult` type exported from `truehuman`. It contains the computed bot likelihood score along with all signals collected during analysis.

## With Plugins

TrueHuman supports an optional `plugins` array that extends the default signal collection with additional third-party verification layers. The example below demonstrates how to include the `grecaptcha` plugin, which validates the current visitor against Google's reCAPTCHA service and sends the resulting token to a backend endpoint for server-side verification.

Because plugin-based analysis is asynchronous, the hook uses an immediately invoked async function expression (IIFE) inside `useEffect` to handle the `await` properly.

```tsx
import { useEffect, useState } from "react"
import { analyze, detector, type AnalyzeResult } from "truehuman"

export function useBotDetectionWithRecaptcha(siteKey: string) {
  const [result, setResult] = useState<AnalyzeResult | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    (async () => {
      const res = await analyze({
        plugins: [
          detector.grecaptcha({
            siteKey,
            endpoint: "/api/verify-recaptcha",
          }),
        ],
      })
      setResult(res)
      setLoading(false)
    })()
  }, [siteKey])

  return { result, loading }
}
```

### Plugin Configuration

The `detector.grecaptcha` plugin accepts the following options:

| Option | Type | Description |
|---|---|---|
| `siteKey` | `string` | Your Google reCAPTCHA v3 site key, obtained from the Google reCAPTCHA admin console |
| `endpoint` | `string` | The path to your server-side verification endpoint, which receives and validates the reCAPTCHA token |

The `siteKey` is listed as a dependency of the `useEffect`, so the hook will re-run analysis automatically if the site key ever changes, for example when switching between environments.

### Server-Side Endpoint

The `endpoint` you provide receives the reCAPTCHA token generated by the client. Your backend is responsible for exchanging that token with Google's verification API and returning a response indicating whether the visitor passed. A minimal Express.js example is shown below.

```ts
import express from "express"

const app = express()
app.use(express.json())

app.post("/api/verify-recaptcha", async (req, res) => {
  const { token } = req.body
  const response = await fetch(
    `https://www.google.com/recaptcha/api/siteverify?secret=${process.env.RECAPTCHA_SECRET}&response=${token}`,
    { method: "POST" }
  )
  const data = await response.json()
  res.json({ success: data.success, score: data.score })
})
```

Keep your reCAPTCHA secret key server-side only and never expose it to the browser.
